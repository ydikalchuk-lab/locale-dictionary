# M2: Таблиця + пошук + переклад

## Завдання
Реалізувати фронтенд: таблиця локалей, пошук у реальному часі, перемикання мови інтерфейсу.

## Кроки
1. **`index.html`** — повна розмітка:
   - Шапка: заголовок, перемикач EN/UA
   - Поле пошуку з placeholder (залежить від мови)
   - Контейнер `<div id="table-container">` для таблиці
   - Скелетон-заглушка (5 рядків)

2. **`style.css`** — стилізація:
   - Базова типографія, кольори (світла тема або темна — вирішимо потім)
   - Таблиця: мінімальні відступи, рамки, hover ефект
   - Пошук: стилізоване поле
   - Скелетон: анімовані сірі блоки
   - Адаптивність (горизонтальний скрол на мобільному)

3. **`i18n.js`** — словник перекладів:
   - Об'єкт з ключами для всіх текстів UI
   - Поточна мова в localStorage
   - Функція оновлення тексту на сторінці при зміні мови

4. **`app.js`** — логіка:
   - `fetch('/api/locales')` при завантаженні
   - Рендер таблиці (прапор, код, мова, країна, валюта, TLD, кнопка "Деталі")
   - Фільтрація рядків при вводі в пошук (по 6 полях)
   - Оновлення таблиці при зміні мови (заголовки колонок, тексти)
   - Стани: loading (skeleton), error (повідомлення), empty ("нічого не знайдено")

## Фініш
1. `http://localhost:3000` → таблиця з усіма локалями, колонки: 🇦🇺 | en-AU | English | Australia | AUD | .au | [Деталі]
2. Введення "au" в пошук → фільтрація до Australia, Austria, etc.
3. Клік EN/UA → заголовки колонок і тексти змінюються, дані локалей залишаються англійськими
4. Скелетон видно під час завантаження
5. Якщо відключити сервер — повідомлення про помилку
6. Пошук без результатів → "Нічого не знайдено / Nothing found"
